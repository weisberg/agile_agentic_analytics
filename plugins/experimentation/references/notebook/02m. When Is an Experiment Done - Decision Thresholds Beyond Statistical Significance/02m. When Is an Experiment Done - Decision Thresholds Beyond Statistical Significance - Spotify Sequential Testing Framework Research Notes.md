# Spotify Sequential Testing Framework Research Notes

## Source
Spotify Engineering Blog - "Choosing a Sequential Testing Framework" (March 2023)
Authors: Mårten Schultzberg and Sebastian Ankargren

## Key Decision Criteria

Spotify identifies two main parameters for choosing sequential testing approach:
1. **Data infrastructure**: Batch vs streaming data delivery
2. **Sample size estimation**: Ability to estimate maximum sample size in advance

## Sequential Testing Methods Compared

### 1. Group Sequential Tests (GST) - Spotify's Choice
- Uses alpha spending approach (Lan and DeMets 1983)
- **Pros**:
  - Flexible alpha spending over time
  - Only spend alpha when you peek
  - Easy to explain (related to z-tests)
  - Highest power when max sample size can be estimated
- **Cons**:
  - Requires knowing/estimating max sample size in advance
  - Undersampling → conservative (lower false positive rate)
  - Oversampling → inflated false positive rate
  - Not feasible for streaming (>few hundred analyses)
  - Requires numerical integration for critical values

### 2. Always Valid Inference (AVI)
Two variants: mSPRT and GAVI

**mSPRT (mixture Sequential Probability Ratio Test)**
- Used by: Optimizely, Uber, Netflix, Amplitude

**GAVI (Generalized Always Valid Inference)**
- Proposed by Howard et al. (2021)
- Used by: Eppo

**Pros**:
- Easy to implement
- No expected sample size required
- Allows unlimited sampling
- Arbitrary stopping rules
- Supports both streaming and batch data

**Cons**:
- Requires choosing mixing distribution parameters (non-trivial)
- Less power with batch data vs streaming
- Harder to understand for traditional statistics practitioners
- Conservative when not testing after each observation

### 3. Corrected-Alpha Approach (CAA)
- Used by: Statsig
- **Major issue**: Does NOT bound false positive rate
- Not considered proper sequential test by Spotify
- False positive rate depends on sample size and number of analyses

### 4. Bonferroni Corrections (Baseline)
- **Pros**: Easy to implement and explain
- **Cons**: 
  - Must decide max analyses in advance
  - Highly conservative with many analyses
  - Low power

## Spotify's Decision Rationale

**Why GST works for Spotify:**
- Data arrives in daily batches
- Most experiments run for few weeks
- Can estimate sample sizes reasonably well
- Typical experiments have <few hundred analyses
- GST provides highest power in this setting

**When streaming data is available:**
- AVI tests (mSPRT/GAVI) can be computed after each observation
- Utilize full potential with streaming
- Can handle batch but becomes conservative

## Risk Management Framework

**Two key risks in experimentation:**
1. **False positives**: Shipping changes without positive impact
2. **False negatives**: Missing changes that improve user experience

**The "peeking problem":**
- Standard tests (z-test, t-test) only control risk if used once
- Repeated testing inflates false positive rate
- Example: 5% intended rate → ~10% actual with 2 tests
- Rate grows quickly with more peeks

## Simulation Study Findings

**Setup:**
- 100,000 replications per parameter combination
- Normal distribution (mean=1, variance=1)
- 500 observations per group (balanced)
- One-sided tests, alpha=5%
- Varied intermittent analyses: 14, 28, 42, 56, or streaming (500)

**Key Results:**
- All tests except oversampled GST and CAA successfully bound false positive rate
- AVI tests conservative when not performed after each observation
- GST with correct sample size maintains intended false positive rate
- Oversampled GST inflates false positive rate
- CAA inflates false positive rate with any peeking

## Practical Implications

1. **For batch data + known sample size**: GST optimal
2. **For streaming data**: AVI tests (mSPRT/GAVI) optimal
3. **For unknown sample size**: AVI tests safer choice
4. **Avoid**: CAA for sequential testing (doesn't solve peeking problem)

## Alpha Spending Functions
- Quadratic vs cubic power family
- Choice affects power when undersampling
- Less critical when reaching planned sample size

## Industry Adoption Patterns
- **Spotify**: Group Sequential Tests
- **Optimizely, Uber, Netflix, Amplitude**: mSPRT
- **Eppo**: GAVI
- **Statsig**: Corrected-Alpha Approach (controversial)
