# Minimum Detectable Effect and Power Analysis

## Source: Statsig - MDE Impact (Dec 2024)

### Definition
**Minimum Detectable Effect (MDE):** The smallest change in a metric that your experiment can detect with statistical significance. Like setting the sensitivity dial on your experiment.

### Core Trade-off

**Lower MDE (higher sensitivity):**
- Detects tiny changes
- Requires larger sample size
- More traffic needed
- Longer duration
- Higher cost

**Higher MDE (lower sensitivity):**
- Requires smaller sample size
- Faster experiments
- Lower cost
- Risk: Missing smaller but meaningful improvements

### Calculation Example

**Baseline:** 5% conversion rate
**MDE set at 10%:** Detecting 0.5 percentage point change

**Sample size requirements:**
- 10% MDE: ~2,922 total conversions needed
- 5% MDE: ~11,141 total conversions needed

**Key insight:** Halving the MDE nearly quadruples the required sample size

### Best Practices

**1. Align with business objectives and KPIs**
- Detect changes that matter in real world, not just statistically
- Consider practical significance vs statistical significance

**2. Balance sensitivity with practicality**
- Not worth detecting minuscule changes with little business impact
- Small improvements can add up over time (don't ignore completely)

**3. Consider context and user segments**
- Appropriate MDE varies by user segments
- May differ across platforms
- Tailor MDE settings to specific contexts

**4. Use power analysis tools**
- Input baseline metrics and desired significance levels
- Determine appropriate MDE and sample size
- Make informed design decisions

### Relationship to Experiment Design

**MDE determines:**
- Experiment sensitivity
- Required number of users
- Test duration
- Resource allocation

**Design considerations:**
- Baseline metrics (current conversion rate)
- Desired statistical significance level (typically 95%)
- Desired statistical power (typically 80%)
- Split ratios between variants
- Test type (one-tailed vs two-tailed)

### Strategic Implications

**Business goal alignment:**
Must answer: "What size change would make a real difference to our organization?"

**Resource optimization:**
Avoid investing resources to detect changes that don't matter operationally

**Opportunity cost:**
Missing small meaningful improvements vs cost of longer experiments

**Segment-specific MDEs:**
Different user groups may warrant different sensitivity levels based on their value or behavior patterns
