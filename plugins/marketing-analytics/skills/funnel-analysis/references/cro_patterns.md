# CRO Patterns & Hypothesis Templates

Common conversion rate optimization patterns and structured hypothesis
templates organized by funnel stage.

## Hypothesis Template Structure

Every CRO hypothesis generated by the funnel-analysis skill follows this
structure:

```
Observation:  [What the data shows — e.g., "42% drop-off at checkout stage"]
Hypothesis:   [Why we believe this happens — e.g., "Users abandon because
               shipping cost is revealed late"]
Test Type:    [A/B test, multivariate, redirect, etc.]
Change:       [Specific intervention — e.g., "Show estimated shipping cost
               on product page"]
Primary KPI:  [Metric to measure — e.g., "Checkout completion rate"]
Guardrails:   [Metrics that must not degrade — e.g., "AOV, return rate"]
Expected Lift:[Conservative estimate — e.g., "3-7% relative improvement"]
Priority:     [Score from bottleneck ranking]
```

## Patterns by Funnel Stage

### Entry / Landing Stage

| Pattern | Signal | Hypothesis Template |
| :------ | :----- | :------------------ |
| High bounce rate | >60% single-page sessions | Page content does not match traffic source intent. Test headline and hero content alignment with top referring campaigns. |
| Slow load time correlation | Bounce rate increases with page load time | Performance is causing abandonment. Test reduced page weight, lazy-loaded images, or AMP version. |
| Mobile drop-off gap | Mobile conversion 50%+ lower than desktop | Mobile experience has usability issues. Test simplified mobile layout, larger tap targets, sticky CTA. |
| Scroll depth cliff | <30% of users scroll past first viewport | Above-fold content fails to communicate value. Test value proposition placement, social proof above fold. |

### Registration / Signup Stage

| Pattern | Signal | Hypothesis Template |
| :------ | :----- | :------------------ |
| Form abandonment | >40% start but do not submit | Form is too long or asks for sensitive info too early. Test progressive disclosure or reduced field count. |
| Social login gap | Sites without social login show higher drop-off | Account creation friction is too high. Test adding Google/Apple SSO options. |
| Password requirements | Drop-off spikes at password field | Password complexity rules frustrate users. Test passwordless authentication or relaxed requirements with strength meter. |
| Email verification wall | Large drop between signup and verification | Verification step loses momentum. Test delayed verification (verify later) or magic link. |

### Consideration / Product Evaluation Stage

| Pattern | Signal | Hypothesis Template |
| :------ | :----- | :------------------ |
| Comparison paralysis | Users view 5+ product pages without adding to cart | Too many options without clear differentiation. Test comparison tables, "most popular" badges, or guided recommendations. |
| Price page drop-off | High exit rate on pricing page | Pricing is unclear or perceived as too high. Test price anchoring, annual vs monthly toggle default, or feature-value framing. |
| Missing social proof | Low conversion on pages without reviews | Users lack confidence in product quality. Test adding reviews, ratings, case studies, or trust badges. |
| Information gap | High drop-off on detail-sparse pages | Users cannot make informed decisions. Test richer product descriptions, FAQs, or demo videos. |

### Cart / Add-to-Cart Stage

| Pattern | Signal | Hypothesis Template |
| :------ | :----- | :------------------ |
| Cart abandonment | >70% of carts are abandoned | Cart experience has friction or commitment anxiety. Test persistent cart, exit-intent offers, or cart recovery emails. |
| Shipping cost surprise | Drop-off spikes when shipping is calculated | Unexpected costs cause abandonment. Test showing shipping estimate earlier or free-shipping threshold. |
| Missing urgency | Long dwell time in cart without checkout | Users are not motivated to complete now. Test low-stock indicators, limited-time offers, or countdown timers. |
| Guest checkout absence | Drop-off at "create account" prompt in checkout | Forced registration blocks conversion. Test guest checkout option. |

### Checkout / Payment Stage

| Pattern | Signal | Hypothesis Template |
| :------ | :----- | :------------------ |
| Payment method gap | Drop-off at payment selection | Preferred payment method is unavailable. Test adding payment options (PayPal, Apple Pay, BNPL). |
| Trust deficit | Drop-off at credit card entry | Users do not trust the payment process. Test security badges, SSL indicators, money-back guarantee placement. |
| Form complexity | Multi-page checkout has higher drop-off than single | Too many steps cause fatigue. Test single-page checkout, progress indicator, or accordion checkout. |
| Error handling | Drop-off after form validation errors | Error messages are unclear or frustrating. Test inline validation, clearer error copy, and auto-formatting. |

### Post-Purchase / Activation Stage

| Pattern | Signal | Hypothesis Template |
| :------ | :----- | :------------------ |
| Onboarding drop-off | >50% of signups never complete setup | Onboarding is too complex or value is not demonstrated quickly. Test simplified onboarding, quick-win first task, or onboarding checklist. |
| Feature discovery gap | Users do not engage with core features in first session | Key features are not discoverable. Test guided tours, contextual tooltips, or "getting started" email series. |
| Aha-moment delay | Users who reach value milestone retain 3x better | Time to value is too long. Test streamlined paths to the aha moment, pre-populated templates, or sample data. |

## Financial Services Specific Patterns

| Pattern | Signal | Hypothesis Template |
| :------ | :----- | :------------------ |
| KYC abandonment | >50% drop-off at identity verification | Verification is perceived as too invasive or complex. Test progress indicators, explain-why tooltips, document upload UX improvements. Do NOT remove required steps. |
| Suitability fatigue | Drop-off during risk questionnaire | Questionnaire feels burdensome. Test conversational format, progress bar, or save-and-resume. Do NOT reduce required questions. |
| Disclosure overload | Drop-off at regulatory disclosure pages | Wall of text overwhelms users. Test accordion disclosures, summary-then-detail format, or highlighted key terms. All disclosure content must remain present. |
| Funding friction | Drop-off at account funding step | Linking external accounts is cumbersome. Test Plaid integration, micro-deposit alternatives, or delayed funding option. |

## Priority Scoring for Hypotheses

Hypotheses inherit their priority from the bottleneck score of the stage they
address. Within a stage, rank hypotheses by:

1. **Evidence strength:** Stronger data signals (larger sample, clearer
   pattern) rank higher.
2. **Implementation effort:** Lower-effort changes rank higher when impact
   estimates are similar.
3. **Risk level:** Lower-risk changes (no regulatory implications, no
   engineering dependencies) rank higher.

Composite hypothesis priority:

```
hypothesis_priority = bottleneck_score * evidence_strength * (1 / effort_level)
```

Where `evidence_strength` is rated 1-3 and `effort_level` is rated 1-3
(1 = low effort).
